## 0.5 hr: IoU, GIoU, DIoU, CIoU, mAP (A2-00-IoU-and-mAP.ipynb)

**Papers:** [GIoU](https://arxiv.org/abs/1902.09630) — *Generalized Intersection over Union: A Metric and A Loss for Bounding Box Regression* · [DIoU/CIoU](https://arxiv.org/abs/1911.08287)

### Why start here

> "Before we talk about any detection model, we need to agree on what 'correct' means. If a model draws a box around a dog — how do we decide if that's a good box or a bad one?"

Everything in detection — anchor assignment, NMS, training loss, evaluation — is built on top of one concept: **IoU**.

---

### IoU

$$\text{IoU}(A, B) = \frac{|A \cap B|}{|A \cup B|}$$

- Range: 0 (no overlap) → 1 (perfect match)
- Pascal VOC: IoU ≥ 0.5 = correct detection
- COCO: average over IoU 0.5–0.95 — much stricter

**Quick code demo:**
```python
def iou(boxA, boxB):
    xA = max(boxA[0], boxB[0]);  yA = max(boxA[1], boxB[1])
    xB = min(boxA[2], boxB[2]);  yB = min(boxA[3], boxB[3])
    inter = max(0, xB - xA) * max(0, yB - yA)
    areaA = (boxA[2]-boxA[0]) * (boxA[3]-boxA[1])
    areaB = (boxB[2]-boxB[0]) * (boxB[3]-boxB[1])
    return inter / (areaA + areaB - inter)
```

**Pause and ask:** "Two boxes have IoU = 0. What's the gradient of `L = 1 - IoU`?"
> Zero — optimizer has no signal. This is IoU's fundamental problem as a loss function.

---

### GIoU → DIoU → CIoU

| Metric | What it adds | Fixes |
|--------|-------------|-------|
| **IoU** | Intersection/Union | — |
| **GIoU** | Penalizes wasted space in enclosing box | Zero gradient when no overlap |
| **DIoU** | Penalizes center-point distance | Slow convergence when box inside GT |
| **CIoU** | Adds aspect ratio consistency | All three geometric factors |

$$\text{GIoU} = \text{IoU} - \frac{|C \setminus (A \cup B)|}{|C|}$$

When boxes don't overlap, IoU = 0 but GIoU < 0 — gradient still flows.

**DIoU-NMS:** In crowded scenes, two people can have high IoU but different centers. Standard NMS suppresses one incorrectly. DIoU-NMS is less aggressive.

---

### mAP

```
1. For each class, rank predictions by confidence (high → low)
2. For each prediction: IoU ≥ threshold + unmatched → TP, else → FP
3. Compute Precision-Recall curve
4. AP = area under PR curve
5. mAP = mean AP over all classes
```

- **VOC mAP**: single threshold (0.5), 11-point interpolation
- **COCO mAP**: average over 10 thresholds (0.5:0.05:0.95) — much stricter

### 1.1 R-CNN (2014) — Cells 4–5

**Paper:** [Rich Feature Hierarchies for Accurate Object Detection](https://arxiv.org/abs/1311.2524)

**Pipeline:**
```
Image → Selective Search (~2,000 boxes)
      → warp each to 227×227 → AlexNet → 4096-dim feature
      → SVM classifier → class label
      → Bbox regressor → refined (x,y,w,h)
```

**Selective Search** — classical algorithm (not learned) that groups similar pixels into ~2,000 plausible candidate boxes per image.

**3 separate losses — trained independently:**
1. CNN fine-tune → softmax cross-entropy
2. SVM → hinge loss per class
3. Bbox regressor → MSE on log-space offsets: $t_x = (G_x - P_x)/P_w$, $t_w = \log(G_w/P_w)$

**Why 3 separate?** Can't backprop through RoI warping — no end-to-end training possible.

**Code demo (Cell 5):** visualizes ~2,000 proposals on `dog-cycle-car.png` + 6 sample warped crops.

---

### 1.2 Fast R-CNN (2015) — Cells 6–7

**Paper:** [Fast R-CNN](https://arxiv.org/abs/1504.08083)

**Key insight:** Run CNN once on the whole image, extract all regions from the shared feature map.

**RoI Pooling:** divide each proposal's feature region into 7×7 grid, max-pool each cell → fixed-length feature regardless of proposal size.

**Multi-task loss — first end-to-end detector:**
$$\mathcal{L} = \mathcal{L}_{\text{cls}}(p, u) + \lambda \cdot \mathbf{1}[u \geq 1] \cdot \mathcal{L}_{\text{loc}}(t^u, v)$$

- $\mathcal{L}_{\text{cls}} = -\log p_u$ — cross-entropy over K+1 classes
- $\mathcal{L}_{\text{loc}}$ — smooth L1 (Huber) on box offsets, **foreground only** ($u \geq 1$)
- $\mathbf{1}[u \geq 1]$ — skip bbox loss for background (no GT box to regress)

9× faster training, 213× faster testing. Still bottlenecked by Selective Search (~2s CPU).

**Code demo (Cell 7):** backbone runs once → shared feature map → RoI Pool per proposal.

---

### 1.3 Faster R-CNN (2016) — Cells 8–9

**Paper:** [Faster R-CNN: Towards Real-Time Object Detection with RPN](https://arxiv.org/abs/1506.01497)

**Key insight:** The feature map already knows where things are — train a small network on it to predict proposals instead of Selective Search.

**Anchors:** at each feature map location, RPN considers k=9 shapes (3 scales × 3 ratios). Positive: IoU > 0.7, Negative: IoU < 0.3.

**Two losses trained jointly:**

RPN loss:
$$\mathcal{L}_{\text{RPN}} = \frac{1}{N_{\text{cls}}} \sum_i \mathcal{L}_{\text{cls}}(p_i, p_i^*) + \lambda \frac{1}{N_{\text{reg}}} \sum_i p_i^* \cdot \mathcal{L}_{\text{reg}}(t_i, t_i^*)$$

Detection head loss (same as Fast R-CNN):
$$\mathcal{L}_{\text{det}} = \mathcal{L}_{\text{cls}}(p, u) + \lambda \cdot \mathbf{1}[u \geq 1] \cdot \mathcal{L}_{\text{loc}}(t^u, v)$$

Both losses backprop through the shared backbone simultaneously.

**Code demo (Cell 9):** shows 9 anchor shapes at one location, all valid anchors across feature map, top-300 proposals after NMS.

| Model | Proposals | Speed | Loss |
|---|---|---|---|
| R-CNN | Selective Search + 2000× CNN | ~47s | 3 separate |
| Fast R-CNN | Selective Search + 1× CNN | ~2s | 1 joint (cls + smooth L1) |
| Faster R-CNN | RPN on GPU (~10ms) | ~0.2s | 2 joint (RPN + det) |

## 1 hr: Image Segmentation (A2-02-Image-Segmentation.ipynb)

### The framing

> "Detection gives you a box. For a surgeon, a box around a tumor is useless — they need to know *which exact pixels*. That's segmentation."

| Task | Output | Example |
|------|--------|---------|
| Detection | Bounding box | Where is the car? |
| Semantic segmentation | Per-pixel class | Which pixels are road vs sky? |
| Instance segmentation | Per-pixel + per-object | Which pixels belong to *this* car vs *that* car? |

---

### Mask R-CNN (Cells 3–4)

**Paper:** [Mask R-CNN](https://arxiv.org/abs/1703.06870)

Faster R-CNN + mask head: for every detected box, also predict a 28×28 binary mask.

**RoI Align vs RoI Pooling:**
> "RoI Pooling rounds to integer pixels — 1px misalignment. Fine for boxes. Terrible for masks. RoI Align uses bilinear interpolation, no rounding."

**Loss:** $\mathcal{L} = \mathcal{L}_{\text{cls}} + \mathcal{L}_{\text{box}} + \mathcal{L}_{\text{mask}}$ — $\mathcal{L}_{\text{mask}}$ is binary CE applied only to GT class mask.

**Code demo (Cell 4):** runs pretrained `maskrcnn_resnet50_fpn` (COCO weights) on `dog-cycle-car.png`. Shows 3 panels: original → bounding boxes → instance masks with contours.

---

### FCN → U-Net (Cells 2, 8–9)

**U-Net architecture diagram is in Cell 2** — walk through it:

- **Left (encoder)**: DoubleConv + MaxPool, channels 1→64→128→256→512
- **Bottom (bottleneck)**: 1024ch, smallest spatial size, no skip connection
- **Right (decoder)**: up-conv 2×2 → concat skip → DoubleConv
- **Gray arrows (copy and crop)**: skip connections — bring high-res encoder features to decoder
- **Top right**: Conv 1×1 → n_classes

**Why skip connections matter more for segmentation:**
> "Classification only needs *what*. Segmentation needs *what* AND *exactly where*. Without skip connections the decoder tries to reconstruct exact pixel boundaries from a blurry bottleneck."

**Two model variants:**

| Model | Encoder | Skip connections | Pretrained |
|---|---|---|---|
| `UNet` (Cell 9) | Custom DoubleConv | ✅ | ❌ |
| `UNetResNet18` (Cell 11) | ResNet-18 | ✅ | ✅ ImageNet |

Switch with `MODEL_TYPE = 'scratch'` or `'resnet18'` in Cell 13.

**UNetResNet18 decoder path (5 upsamples):**
```
Input 128×128 → stem_conv → H/2 (s0, skip)
              → stem_pool → H/4
              → layer1    → H/4  (s1, skip)
              → layer2    → H/8  (s2, skip)
              → layer3    → H/16 (s3, skip)
              → layer4    → H/32 (s4, skip)
              → bottleneck → H/32
Decoder: H/32→H/16→H/8→H/4→H/2→H (5 upsamples, each concat skip)
```

---

### mIoU

$$\text{mIoU} = \frac{1}{C}\sum_c \frac{TP_c}{TP_c + FP_c + FN_c}$$

> "Why not accuracy? 90% background → predicting all-background = 90% accuracy. mIoU penalizes this hard."

## The Big Picture — Full Timeline

```
2014  R-CNN         "CNN for detection. 3 separate models, 47s/image."
        ↓
2015  Fast R-CNN    "Shared backbone + joint loss. 2s/image."
2015  FCN           "Replace FC → 1×1 conv. First pixel-level output."
2015  U-Net         "Skip connections recover spatial detail."
        ↓
2016  Faster R-CNN  "RPN replaces Selective Search. 0.2s/image."
2016  YOLO v1       "Skip proposals entirely. Single-shot, real-time."
        ↓
2017  Mask R-CNN    "Faster R-CNN + mask head = instance segmentation."
        ↓
2018  YOLOv3        "Multi-scale detection. 3 anchor scales. 500k iterations."
        ↓
2020  GIoU/DIoU/CIoU "Better box regression losses. CIoU now standard in YOLOv4+."
        ↓
2021+ DETR, DINO    "Transformers for detection. No anchors, no NMS."
```

**Closing question:**

> "A hospital wants pixel-level tumor outlines from MRI scans. 300 labeled scans. Which model and training strategy?"
>
> Answer: U-Net with pretrained ResNet-18 encoder. YOLO gives boxes not masks. 300 samples → must use pretrained encoder. Strategy: freeze encoder → train decoder → unfreeze all.
